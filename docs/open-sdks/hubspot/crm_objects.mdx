## Integrating with Hubspot OpenSDK 

Welcome to the Hubspot OpenSDK documentation. OpenSDKs provides a type-safe and standardized way to interact with various APIs, ensuring a consistent and efficient integration experience.

The Hubspot OpenSDK allows you to easily integrate with the Hubspot API, providing a robust set of tools to manage your API interactions with managed auth by OpenInt. 

### Available Methods:

| Method | Path | Description |
|--------|------|-------------|
| **POST** | /crm/v3/objects/{objectType}/batch/read | Read a batch of objects by internal ID, or unique property values
| **GET** | /crm/v3/objects/{objectType}/{objectId} | Read
| **DELETE** | /crm/v3/objects/{objectType}/{objectId} | Archive
| **PATCH** | /crm/v3/objects/{objectType}/{objectId} | Update
| **POST** | /crm/v3/objects/{objectType}/merge | Merge two objects with same type
| **POST** | /crm/v3/objects/{objectType}/batch/archive | Archive a batch of objects by ID
| **PUT** | /crm/v3/objects/{objectType}/{objectId}/associations/{toObjectType}/{toObjectId}/{associationType} | Associate an object with another object
| **DELETE** | /crm/v3/objects/{objectType}/{objectId}/associations/{toObjectType}/{toObjectId}/{associationType} | Remove an association between two objects
| **POST** | /crm/v3/objects/{objectType}/batch/create | Create a batch of objects
| **POST** | /crm/v3/objects/{objectType}/batch/update | Update a batch of objects by internal ID, or unique property values
| **GET** | /crm/v3/objects/{objectType}/{objectId}/associations/{toObjectType} | List associations of an object by type
| **POST** | /crm/v3/objects/{objectType}/gdpr-delete | GDPR DELETE
| **GET** | /crm/v3/objects/{objectType} | List
| **POST** | /crm/v3/objects/{objectType} | Create
| **POST** | /crm/v3/objects/{objectType}/search | 


## Code Example

```typescript

    import {initSDK} from '@opensdks/runtime'
    import {hubspotSdkDef} from '@opensdks/sdk-hubspot'

    const hubspot = initSDK(hubspotSdkDef, {
      auth: {
        // for provider-specific auth
        bearer: `${process.env['HUBSPOT_TOKEN']}`,
        // or for passthrough auth via OPENINT_API_KEY & end user resourceId
        openInt: {
          apiKey: `${process.env['OPENINT_API_KEY']}`,
          resourceId: END_USER_RESOURCE_ID,
        }
      }, params: { ... }
    })

    // Read a batch of objects by internal ID, or unique property values
    const res = await hubspot.POST('/crm/v3/objects/{objectType}/batch/read')
```
