# HubSpot Integration (OAuth2)

This guide enables engineers to connect HubSpot to your application using OpenInt's white-labeled integration toolkit. HubSpot is a leading CRM and automation platform supporting a variety of business and ticketing use cases. OpenInt manages authentication using OAuth2, and provides granular scope control and a streamlined data synchronization flow.

## Prerequisites

1. **HubSpot Developer Account**
   - [Sign up for a HubSpot developer account](https://app.hubspot.com/signup-hubspot/developers).
2. **Create a Public App in HubSpot**
   - Follow the [official HubSpot guide for creating a public app](https://developers.hubspot.com/docs/guides/apps/public-apps/overview#create-a-public-app).
3. Record your app's **Client ID** and **Client Secret** (available in the Auth tab after app creation).

<Info>
OpenInt uses OAuth2 for authentication with HubSpot. Ensure your app is set up for OAuth2 flows and your redirect URIs are correctly configured.
</Info>

## Setting Up Your HubSpot App

1. In your HubSpot developer account, navigate to **Apps** and click **Create app**.
2. Fill in the app name, description, and logo as prompted.
3. Proceed to the **Auth** tab. Note your **Client ID** and **Client Secret**.
4. Add your integration's redirect URI (test with http://localhost for development; use HTTPS and a valid domain in production).

For details and screenshots, see the [HubSpot public app creation docs](https://developers.hubspot.com/docs/guides/apps/public-apps/overview#create-a-public-app).

## Configuring OAuth2 with OpenInt

### Authorization URLs

Configure your integration to use the following endpoints:

- **Authorization Request URL:**  `https://app.hubspot.com/oauth/authorize`
- **Token Request URL:** `https://api.hubapi.com/oauth/v1/token`

### Required Scopes

When creating or configuring your HubSpot app, select the necessary OAuth scopes. Scopes define which resources and operations your integration can access.

All available HubSpot OAuth scopes are documented [here](https://developers.hubspot.com/docs/guides/apps/authentication/working-with-oauth#configure-scopes).

## Quickstart: Integrate HubSpot via OpenInt

1. **Set up your HubSpot OAuth app** as described above.
2. In OpenInt, configure your connector with your HubSpot **Client ID**, **Client Secret**, and the supported scope:

```json
{
  "type": "hubspot",
  "clientId": "YOUR_CLIENT_ID",
  "clientSecret": "YOUR_CLIENT_SECRET",
  "scopes": ["crm.objects.contacts.read"],
  "redirectUri": "YOUR_REDIRECT_URI"
}
```

3. Direct your users to the authorization URL generated by OpenInt, or use OpenInt's built-in Connect Portal embedding.
4. Upon successful authorization, OpenInt will handle token exchange and subsequent API calls securely.

For full details on connection data flow, see [Data Flow](/docs/connect/data-flow.mdx).

## Common Pitfalls and Troubleshooting

<Warning>
Ensure that your **redirect URI** is set exactly the same in HubSpot and OpenInt. Mismatches will result in failed OAuth flows.
</Warning>

- **Missing Scopes:** If you request a scope your app or user account doesn’t support, installation will fail. Only request the scopes your integration and HubSpot account have access to.
- **Account Permissions:** Users must have proper permissions in HubSpot to authorize apps. [Super Admin or App Marketplace Access is required.](https://knowledge.hubspot.com/user-management/hubspot-user-permissions-guide#super-admin)
- **Token Expiration:** Access tokens expire frequently; always handle token refresh using the provided refresh_token.
- **Testing on Developer Accounts:** HubSpot does not allow public app installation on developer accounts. Use a [HubSpot test account](https://developers.hubspot.com/docs/getting-started/account-types) for QA.
